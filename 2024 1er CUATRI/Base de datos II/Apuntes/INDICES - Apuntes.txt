INDICES

-Facilitan la obtención de información de una tabla. Una tabla se indexa por un campo o varios.
-Posibilita el acceso directo y rápido haciendo mas eficiente las búsquedas. Sin índice, el sql server debe recorrer de forma secuencial toda la tabla
-Acelera la recuperación de información.
-Optimiza el acceso a los datos, mejora el rendimiento acelerando las consultas y otras operaciones.

SQL SERVER accede a los datos:
1- recorriendo las tablas
2- empleando indices

Desventajas:
-Consume espacio en disco y genera costo de mantenimiento

SQL SERVER permite crear dos TIPOS de INDICES:
1-Agrupados: Similar a una guía telefónica. Los registros con el mismo valor de campo se agrupan juntos. Una tabla solo puede tener UN índice agrupado. Se utilizan para campos por los que se realizan búsquedas con frecuencia.
2-No Agrupados: Es como el índice de un libro, los datos se almacenan en un lugar diferente al del indicia, los punteros indician el lugar de almacenamiento de los elementos indizados en la tabla. Puedo tener varios indices.

Sobre indices:
-Los mas adecuados son los que tienen valores únicos y los que se utilicen con frecuencia.
-Si no se especifica índice, es no agrupado x defecto
-text, ntext e image no se pueden indizar
-Recomendable crear los agrupados antes q los no agrupados. Los primeros modifican el orden físico de los registros.
-Diferencia es q los agrupados están ordenados y almacenados de forma secuencial en función de su clave.
-SQL Server crea indices cuando se crea una pk o unique en una tabla

Crear índice en una tabla, se permiten valores duplicados:
CREATE INDEX index_name
 ON table_name (column1, column2, ...);

Crea un índice único en una tabla:
CREATE UNIQUE INDEX index_name
 ON table_name (column1, column2, ...);

Crear un índice único no agrupado:
CREATE UNIQUE NONCLUSTERED INDEX IX_NC_PresidentNumber 
    ON dbo.Presidents (PresidentNumber) -- specify table and column name
